# ЕГЭ-чекер — демо-версия (для показа друзьям)

Это самостоятельная копия веб-фронтенда (Vite + React + TypeScript) для деплоя на Vercel.

**Важно:** всё работает на стороне браузера — никакого бэкенда, серверов и **токенов не нужно**.
Данные демонстрационные (проверки, тарифы, оплата, профиль — заглушки). Все кнопки и переходы
рабочие, оплата имитируется (деньги не списываются).

## Как развернуть на Vercel

### Вариант 1 — через сайт vercel.com (проще)
1. Залей эту папку `demo/` в репозиторий на GitHub (или весь проект — тогда на шаге 3 укажешь
   корнем папку `demo`).
2. На vercel.com → **Add New… → Project** → импортируй репозиторий.
3. В настройках проекта:
   - **Root Directory**: `demo` (если залит весь проект; если залита только эта папка — оставь по умолчанию);
   - **Framework Preset**: Vite (определится сам);
   - **Build Command**: `npm run build` (по умолчанию);
   - **Output Directory**: `dist` (по умолчанию).
4. **Deploy** — через минуту получишь ссылку вида `https://...vercel.app`, её и шли друзьям.

### Вариант 2 — через консоль (Vercel CLI)
```bash
npm i -g vercel      # один раз
cd demo
vercel               # первый деплой (превью)
vercel --prod        # боевой деплой → публичная ссылка
```

## Локальный запуск (по желанию)
```bash
cd demo
npm install
npm run dev          # откроется на http://localhost:5173
```

## Что внутри
- `src/` — весь код страниц (Проверка, База аргументов, Тарифы, Оформление, Проекты, Счётчик, Профиль, Вход).
- `src/data/knowledgeBase.json` — база аргументов (зашита в сборку).
- `src/lib/projects.ts` — список проектов (замени ссылки/лого на свои перед показом).
- `vercel.json` — конфигурация деплоя.

> Это снимок текущего фронта. Если потом меняешь основной проект (`frontend-web/`),
> скопируй изменения сюда заново, чтобы демо обновилось.
